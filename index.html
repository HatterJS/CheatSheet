META

<!DOCTYPE html> *required
Tells the computer that it’s looking at an HTML page. Put this at the top of your HTML document.

<meta charset="utf-8"> *required
Tells the computer that you are using the UTF-8 character set (which is the character set for most of the world wide web). Guards against your site turning into gibberish.

<title>..</title> *required
Gives your site a title in the top of the browser bar.

<script src = "http://www.example.com/ scripts/myjavascript.js"></script> Links to external JavaScript files via the src attribute.

<link rel = "stylesheet" href = "http://www.example.com/css/stye.css">
Links to external stylesheets via the href attribute. Use the rel attribute to tell the computer that this is a stylesheet.

Example

<!DOCTYPE html>
<html lang="en">
<head>
 <meta charset="UTF-8">
 <meta http-equiv="X-UA-Compatible" content="IE=edge">
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
 <meta name="robots" content="none">
 <title>Cheat Sheet</title>
 <link rel="stylesheet" href="css/style.css" type="text/css">
</head>

TABLES

<table>...</table>
Use this tag to add tables to your site.

<thead>...</thead>
Denotes the table head, where you put your columns names.

<th>...</th>
Use these tags inside your table head to give each column a name.

<tbody>...</tbody>
Denotes the table body, where all the tabular data goes.

<tr>...</tr>
Use this tag to denote a table row.

<td>...</td>
Put each piece of data inside of a data cell, inside of your table row, inside of your table body.

colspan="2"
An attribute that allows you to combine cells in a string.

rowspan="2"
An attribute that allows you to combine cells in a column.

Example

<table>
<caption>List of employees</caption>
<thead>
    <tr>
      <th>Company</th>
      <th>Contact</th>

   </tr>
 </thead>
 <tbody>
   <tr>
      <td>Alfreds Futterkiste</td>

   </tr>
   <tr>

   </tr>

   </tr>
 </tbody>

Semantic

What are Semantic Elements?

 A semantic element clearly describes its meaning to both the browser and the developer.
 Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.
 Examples of semantic elements: <form>, <table>, and <article> - Clearly defines its content.

SECTION and ARTICLE

<section>...</section>
Defines elements in a document, such as chapters, headings, or any other area of the document with the same theme.

<article>...</article>
Oholds content that makes sense on its own such as articles, blogs, and comments.

Generally developers will use <section> to define a theme for the webpage and use <article> to write independent content for that theme. This does not mean that <article> has to be used with <section>.

Example

<section>
<h2>Defines theme</h2>
<article>
    <p>Writes independent content relating to that theme</p>
 </article>
</section>

EMBEDDING MEDIA

<video>...</video>
Allows us to add videos to our website.

<audio>...</audio>
Allows us to implement audio into our website.

<embed ... />
Can be used to implement any type of media.

These elements are universal in that they all use the <src> attribute to link the source of the content. <video> and <audio> requires a closing tag while <embed> is a self-closing tag.

Example

<video src="4kvideo.mp4">video not supported</video>
<audio src="hiphop.mp3"></audio>
<embed src="smile.gif"/>

FIGURE, FIGCAPTION

<figure>...</figure>
The element is used to encapsulate media such as an image, diagram or code snippet.

<figcaption>...</figcaption>
The element is used to describe the media encapsulated within the previous element.

Developers will normally use <figcaption> within the <figure> element to group the media and description. This way, if a developer decides to change the position of the media, the description will follow along with it.

Example

<figure>
<img src="house.jpg">
<figcaption>The image shows the layout of a qwerty keyboard.
</figcaption>
</figure>

ASAID

<aside>...</aside>
The element is used to mark additional information that can enhance another element but isn’t required in order to understand the main content. Usually, this information would be in a sidebar or a location where it doesn’t obstruct the main piece of content. An example of this would be an article that discusses how to take care of a dog and next to the article an ad would appear advertising a dog grooming product.

HEADER and FOOTER

<header>...</header>
The element represents a container for introductory content or a set of navigational links.
Typically contains:
- one or more heading elements <h1>;
- logo or icon;
- authorship information.

<footer>...</footer>
The element defines a footer for a document or section.
Typically contains:
- authorship information;
- copyright information;
- contact information;
- sitemap;
- back to top links;
- related documents.

NAV

<nav>...</nav>
The element defines a set of navigation links.

<figcaption>...</figcaption>
The element is used to describe the media encapsulated within the previous element.

Notice that NOT all links of a document should be inside a <nav> element. The <nav> element is intended only for major block of navigation links.
Browsers, such as screen readers for disabled users, can use this element to determine whether to omit the initial rendering of this content.

Example

<nav>
 <a href="/html/default">
 <a href="/CSS/default">
 <a href="/JS/default">
</nav>

MAIN

<main>...</main>
The <main> tag specifies the main content of a document.

The content inside the <main> element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.

There must not be more than one <main> element in a document. The <main> element must NOT be a descendant of an <article>, <aside>, <footer>, <header>, or <nav> element.

FOR EXAMPLE

<nav>
<section> <aside>
<article>

FLEX-DIRECTION

Flex-direction property controls the direction in which the flex-items are laid along the main axis.

flex-direction: row;

flex-direction: row-reverse;

flex-direction: column;

flex-direction: column-reverse;

JUSTIFY-CONTENT

The justify content property defines how flex items are laid out on the main axis.

justify-content: flex-start;

justify-content: flex-end;

justify-content: center;

justify-content: space-between;

justify-content: space-around;

justify-content: space-evenly;

ALIGN-ITEMS

It defines how flex-items are laid out on the cross axis. This is the difference between the align-items property and justify-content.

align-items: stretch;

align-items: flex-start;

align-items: flex-end;

align-items: center;

align-items: baseline;

ALIGN-CONTENT

It controls how the flex-items are aligned in a multi-line flex container.

align-content: stretch;

align-content: flex-start;

align-content: flex-end;

align-content: center;

align-content: space-between;

align-content: space-around;

FLEX-WRAP

The flex-wrap property specifies whether the flexible items should be moved to another line or not.

flex-wrap: nowrap;

flex-wrap: wrap;

flex-wrap: wrap-reverse;

ORDER (item 2)

Simply put, with the order property you can move a flex-item from one position to another. The default value for the order property is 0. It may take on either negative or positive values.

order: 0;

order: -1;

order: 15;

FLEX-GROW and
FLEX-SHRINK (item 2)

The flex-grow and flex-shrink properties control how much a flex-item should “grow” (extend) if there are extra spaces, or “shrink” if there are no “extra” spaces.

flex-grow: 0;

flex-grow: 1;

flex-shrink: 1;

flex-shrink: 0;

flex-shrink: 2;

ALIGN-SELF (item 2)

Change the position of a single flex-item along the cross-axis, without affecting the neighboring flex-items.

align-self: auto;

align-self: flex-start;

align-self: flex-end;

align-self: center;

align-self: baseline;

align-self: stretch;

Basics

JavaScript Basics

 JavaScript, also abbreviated to JS, is a programming language used in web development. As one of the core technologies of the web alongside HTML and CSS, JavaScript is used to make webpages interactive and to build web apps. Modern web browsers, which adhere to common display standards, support JavaScript through built-in engines without the need for additional plugins.
 When working with files for the web, JavaScript needs to be loaded and run alongside HTML markup. This can be done either inline within an HTML document or in a separate file that the browser will download alongside the HTML document.

ADDING JS to HTML

 JavaScript code generally lives in one or more js files that are referenced within HTML documents.
 To add the JavaScript code into the HTML pages, we can use the <script>.....</script> tag of the HTML that wrap around JavaScript code inside the HTML.
 When using the script tag <script>, we must always use the attribute name and value of type="text/javascript".
 To include an external JavaScript file, we can use the src attribute with the path to your JavaScript file.

Example

<!DOCTYPE html>
<html lang="en">
<head>
 <meta charset="UTF-8">
 <meta http-equiv="X-UA-Compatible" content="IE=edge">
 <meta name="viewport" content="width=device-width, initial-scale=1.0">
 <meta name="robots" content="none">
 <title>Cheat Sheet</title>
 <link rel="stylesheet" href="css/style.css" type="text/css">
</head>
<body>
 <script type="text/javascript" src="js/script.js"></script>
 </body>

DATA TYPES

String - represents textual data - "Hello, World!"
Number - an integer or a floating-point number - 3, 3.1415, 3e-2
Boolean - any of two values: true or false - true, false
Undefined - a data type whose variable is not initialized - let a;
Null - denotes a null value - let a = null;
BigInt - an integer with arbitrary precision (integer values larger than (253-1)) - 900719925124740999n
Symbol - for unique identifiers - let value = Symbol('hello');
Object - key-value pairs of collection of data - let student = { };

OPERATORS

Assignment Operators

= |assignment operator| a = 7;
+= |addition| a+=5;//a=a+5
-= |subtraction| a-=2;//a=a-2
*= |multiplication| a*=3;//a=a*3
/= |division| a/=2;//a=a/2
%= |remainder| a%=7;//a=a%7
**= |exponentiation| a**=4;//a=a**4

Arithmetic Operators

+ - * / |similar to a calculator| a + b;
++ |increments by 1| a++;//a=a+1
-- |decrements by 1| a--;//a=a-1
** |Exponentiation| a**3;//a=a3

Comparison Operators

== |is equal| 2=='2';//true
=== |identical (equal and of same type)| 7==='7';//false
!= |not equal| 10!=15//true
!== |not identical| 11!=='11'//false
> |greater than| 7>1;//true
< |less than| 4<2;//false
>= |greater than or equal| 5>=3;//true
<= |less than or equal| 7<=7;//true

Logical Operators

&& |logical AND| (2==2&&7=7);//true
|| |logical OR| (3==1||5==5);//true
! |logical NOT| !(9==3)//true
